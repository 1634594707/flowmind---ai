# FlowMind 实施路线图

## 🎯 总体策略

采用 **AI 辅助 + 敏捷迭代** 的开发模式，每 2 周一个 Sprint，快速验证核心价值。

### 目标

## 💡 AI 使用建议

### 每日开发流程

1. **早上**：使用 AI 生成当天的任务代码框架
2. **开发中**：使用 AI 辅助调试和优化
3. **晚上**：使用 AI 生成测试用例和文档

### AI 提示词模板库

创建一个 `prompts/` 目录，保存常用的提示词：

```
prompts/
├── backend/
│   ├── create-module.md
│   ├── create-api.md
│   └── create-test.md
├── frontend/
│   ├── create-page.md
│   ├── create-component.md
│   └── create-hook.md
└── database/
    ├── design-model.md
    └── optimize-query.md
```

### 代码审查清单

每次提交前，使用 AI 检查：
- [ ] 代码规范
- [ ] 安全漏洞
- [ ] 性能问题
- [ ] 测试覆盖

## 📊 进度跟踪

使用 GitHub Projects 或 Jira 跟踪进度：

- **To Do**：待开发任务
- **In Progress**：开发中
- **In Review**：代码审查
- **Done**：已完成

每周五进行 Sprint 回顾，调整下周计划。
